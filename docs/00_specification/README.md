# 00 — Specyfikacja (źródło prawdy)

Dokumenty opisujące, **co system ma robić**, oraz materiały źródłowe, na których się opieramy.
To najbardziej stabilna warstwa dokumentacji — zmiany tu są rzadkie i przemyślane.

Co tu trafia:
- specyfikacja funkcjonalna / wymagania (co system *ma* robić),
- materiały źródłowe i referencyjne, na których opieramy projekt.

**Nie umieszczaj tu** opisu bieżącej implementacji (→ `01_architecture/`) ani planów wdrożenia
(→ `02_plans/`).
