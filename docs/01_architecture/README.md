# 01 — Architektura (stan obecny)

Dokumenty opisujące, **jak system jest faktycznie zbudowany** w danym momencie. Powinny być
aktualizowane wraz z rozwojem kodu, tak aby zawsze odzwierciedlały rzeczywistość.

Co tu trafia:
- architektura modułów, komponentów i przepływów danych,
- opis kluczowych interfejsów / API faktycznie wystawionych przez system.

**Różnica względem `03_reports/`:** tu opisujemy *trwały, aktualny stan*; raporty to *jednorazowe
migawki* analizy z konkretnego momentu.
